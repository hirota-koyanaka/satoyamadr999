# Supabase接続文字列の構築方法

## 接続文字列の形式

SupabaseのPostgreSQL接続文字列は以下の形式です：

```
postgresql://postgres:[PASSWORD]@db.[PROJECT_REF].supabase.co:5432/postgres
```

## 現在のプロジェクト情報

- **プロジェクトID**: `skbvryjtjxmdslfwivqh`
- **接続文字列のテンプレート**: 
  ```
  postgresql://postgres:[PASSWORD]@db.skbvryjtjxmdslfwivqh.supabase.co:5432/postgres
  ```

## パスワードの取得方法

### 方法1: プロジェクト作成時のパスワードを使用

CLIでプロジェクトを作成した際に、パスワードが自動生成されましたが、表示されていません。
以下のコマンドでパスワードをリセットできます。

### 方法2: Supabaseダッシュボードでパスワードをリセット

1. **Supabaseダッシュボードにアクセス**
   - https://supabase.com/dashboard/project/skbvryjtjxmdslfwivqh/settings/database

2. **パスワードをリセット**
   - 「Reset database password」または「Database password」セクションを探す
   - 新しいパスワードを設定（**メモしておく**）

3. **接続文字列を完成させる**
   - パスワードを取得したら、以下の形式で接続文字列を作成：
     ```
     postgresql://postgres:your-password@db.skbvryjtjxmdslfwivqh.supabase.co:5432/postgres
     ```

## 接続文字列を使用する

接続文字列を取得したら、環境変数に設定します：

```bash
export DATABASE_URL="postgresql://postgres:your-password@db.skbvryjtjxmdslfwivqh.supabase.co:5432/postgres"
```

その後、Djangoマイグレーションを実行：

```bash
source venv/bin/activate
python manage.py migrate
```

## 注意事項

- パスワードは機密情報です。Gitにコミットしないでください
- 接続文字列に特殊文字が含まれる場合は、URLエンコードが必要な場合があります
- パスワードに `@` や `:` などの特殊文字が含まれる場合は、URLエンコードが必要です
